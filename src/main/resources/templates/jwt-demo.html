JWT Refresh Token Lifecycle Demo

Mô phỏng vòng đời thực tế của access + refresh token. Mỗi nút bấm = 1 bước, log chi tiết vào F12 Console và ô log bên dưới.

Server config: access TTL = 5 phút, refresh TTL = 7 ngày. Demo user: alice / wonderland

1. Trạng thái phiên

Access token

Status: CHƯA CÓ

Hết hạn sau: —

(chưa login)
Payload đã decode: 
—

Refresh token

Status: CHƯA CÓ

Đếm số lần xoay vòng: 0

(chưa login)
Token này lưu trong memory, F5 sẽ mất.

2. Mô phỏng vòng đời (bấm theo thứ tự)

Bước A — T+0:00 Login

Bước B — T+0:15 Gọi API protected (token còn hạn)

Bước C — T+5:30 Access token hết hạn → gọi API → 401

Bước D — T+5:31 Auto-refresh

Bước E — Gọi lại API với token mới

Bước F — Auto-mode: bật vòng lặp tự gọi /api/me mỗi 8s (sẽ proactive-refresh khi sắp hết hạn, hoặc reactive-refresh khi nhận 401)

Để bật/tắt vòng lặp gọi tự động — minh hoạ refresh "vô hình" cho user.

3. Mô phỏng tấn công — Refresh Token Reuse

Giả sử attacker đánh cắp refresh token CŨ (đã bị rotate). Khi attacker dùng → server phát hiện reuse → revoke cả family.

Sau H, cả family bị revoke. User thật cũng bị đá ra — phải login lại.
